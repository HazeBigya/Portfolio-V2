# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- `npm run dev` — Vite dev server at http://localhost:5173 (HMR).
- `npm run build` — `react-router build` (Vite client + SSR bundles → `build/client/` and `build/server/`).
- `npm run start` — `react-router-serve ./build/server/index.js`. Default port 3000 (override with `PORT=…`).
- `npm run typecheck` — `react-router typegen && tsc` (no emit). `typegen` regenerates `.react-router/types/**` which back the `./+types/<route>` virtual imports.
- No lint, no test runner configured.
- Docker: `docker build -t portfolio . && docker run -p 3000:3000 portfolio`.

## Architecture

React Router v7 (`@react-router/dev` + `@react-router/serve`) on Vite 6, TypeScript strict, Tailwind v4 via `@tailwindcss/vite`. SSR enabled (`react-router.config.ts: ssr: true`). UI built on Radix primitives + shadcn-style components, animated with framer-motion.

### Routing

`app/routes.ts` is the route manifest (config-based, not file-system convention). Add new routes by appending to the exported array, e.g. `route("about", "routes/about.tsx")`. Route module types come from generated `./+types/<routeName>` imports — run `npm run typecheck` (or `react-router typegen`) after adding routes.

### Directory layout (non-standard for shadcn)

`components/`, `hooks/`, and `lib/` live at the **repo root**, NOT under `app/`. Only `app/root.tsx`, `app/routes.ts`, `app/routes/`, and `app/welcome/` are inside `app/`.

- `app/root.tsx` — HTML shell, `Layout`/`App`/`ErrorBoundary` exports, global meta, font links, forces `<html class="dark">`, imports `./app.css`.
- `app/routes/_index.tsx` — landing page composing portfolio sections (hero, experience timeline, contact).
- `components/` — page-level pieces (`site-header`, `background-fx`, `experience-timeline`, `ai-orbs`, `contact-actions`, `section-header`, `theme-provider`).
- `components/ui/` — shadcn primitives wrapping Radix.
- `hooks/`, `lib/utils.ts` (`cn()` clsx + tailwind-merge).
- `app/app.css` — Tailwind v4 source: `@import "tailwindcss"`, `@theme`, `@custom-variant dark`, CSS variable design tokens (oklch). Processed by `@tailwindcss/vite`.
- `public/` — static screenshots.

### Import paths

`tsconfig.json` defines `~/*` → `./app/*`. Because `components/` lives outside `app/`, the `~/` alias does NOT reach it. Existing routes import shared code via relative paths like `../../components/ui/button`. Either keep using relative paths or add a second tsconfig alias if you want a shorter form.

### Styling pipeline

Single source: `app/app.css`, processed by `@tailwindcss/vite`. Tailwind v4 syntax (CSS-first config) — design tokens live in `@theme inline { … }` blocks mapping CSS vars to Tailwind utility namespaces.

`tailwind.config.ts` is a v3-style HSL token file that is NOT loaded (v4 ignores JS config unless explicitly referenced via `@config`). The actual tokens are the oklch CSS vars in `app/app.css`. Do not edit `tailwind.config.ts` expecting it to take effect.

Do NOT add a `sass` step — earlier attempts piped this CSS through `sass`, which mangles Tailwind directives.

### Auxiliary
- `vite-tsconfig-paths` resolves tsconfig path aliases inside Vite.
- `isbot` is a runtime dep used by the default RR v7 entry server (for bot UA detection). No custom `app/entry.server.tsx` / `app/entry.client.tsx` — relies on `@react-router/dev` defaults.
- `.react-router/` is generated (typegen output). Safe to delete; regenerated by typecheck/build.
